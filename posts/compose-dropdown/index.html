Conclusion

We can see the benefit of using composable here. In the old view system, Views has itâ€™s own saved instance state backed by their ids this had few advantages, yet leaves the system unpredictable in many cases. Devs have to take care of keeping the UI and model in sync.

In compose, data drives the UI. Take a look at cityName, initial value set to the component â€” and then menu item click updates the variable name - not the Text/Label view. This is called unidirectional data flow. Data paints the UI, and UI updates the model. No UI vs UI communication happens here. That means, we donâ€™t take value from the UI â€” rather state can give it to us.
